Control Grid

Use nodel-control-grid to divide a page column into equal-width touch control cells. Use nodel-control-space for deliberate blank cells.

Fixed Columns

A grid with columns="3" keeps each child to one third of the parent width. A single child occupies only the first cell.

One third 
<nodel-control-grid columns="3">
  <nodel-button>One third</nodel-button>
</nodel-control-grid>

Wrap

Additional controls wrap naturally into later rows while keeping equal cell widths.

One Two Three Four Five Six 
<nodel-control-grid columns="3">
  <nodel-button>One</nodel-button>
  <nodel-button>Two</nodel-button>
  <nodel-button>Three</nodel-button>
  <nodel-button>Four</nodel-button>
  <nodel-button>Five</nodel-button>
  <nodel-button>Six</nodel-button>
</nodel-control-grid>

Nested And Empty Cells

Nest grids to divide a cell further. Add nodel-control-space when a blank control slot is intentional.

Future tall control area Mute Speech Music 
<nodel-control-grid columns="2">
  <nodel-fader label="Volume"></nodel-fader>
  <nodel-control-grid columns="1">
    <nodel-button>Mute</nodel-button>
    <nodel-button>Speech</nodel-button>
    <nodel-button>Music</nodel-button>
    <nodel-control-space></nodel-control-space>
  </nodel-control-grid>
</nodel-control-grid>

Media

Use nodel-image and nodel-icon as standalone control-grid items or as inline button children. Use nodel-status-indicator inside buttons for compact signal state.

Standalone Media

Direct children of nodel-control-grid occupy their own control cell. Visible labels are optional; use alt for accessible names without visible text. 

<nodel-control-grid columns="2" md="4">
  <nodel-image src="./v2/img/logo.png" alt="Nodel logo" size="lg"></nodel-image>
  <nodel-icon name="power" alt="Power" tone="accent" variant="soft"></nodel-icon>
  <nodel-icon name="volume" label="Volume" tone="success" size="lg" variant="bordered"></nodel-icon>
  <nodel-icon name="warning" label="Warning" tone="warning" size="xl"></nodel-icon>
</nodel-control-grid>

Buttons

Use nodel-button for touch-sized buttons. Buttons without action are inert; action buttons are intended for node pages with relative REST action endpoints.

Variants

Set variant for Bootstrap-like emphasis: primary, success, info, warning, danger, and link-style options.

Default Primary Success Info Warning Danger Ghost Link 
<nodel-control-grid columns="2" md="4">
  <nodel-button>Default</nodel-button>
  <nodel-button variant="primary">Primary</nodel-button>
  <nodel-button variant="success">Success</nodel-button>
  <nodel-button variant="info">Info</nodel-button>
  <nodel-button variant="warning">Warning</nodel-button>
  <nodel-button variant="danger">Danger</nodel-button>
  <nodel-button variant="ghost">Ghost</nodel-button>
  <nodel-button variant="link">Link</nodel-button>
</nodel-control-grid>

Signal State

Button state usually follows Nodel signals. The shorthand signal target controls active; explicit signals targets can control active, disabled, or label. 

<nodel-control-grid columns="3">
  <nodel-button action="SetSource" arg="Chromecast" signal="Source">Chromecast</nodel-button>
  <nodel-button action="SetSource" arg="TV" signal="Source">TV</nodel-button>
  <nodel-button action="StartShow" signals="ShowRunning:active; ControlsLocked:disabled">Start Show</nodel-button>
</nodel-control-grid>

Media And Indicators

Media components render inline when placed inside nodel-button. The same size values apply inline. Indicators keep their own signal binding.

<nodel-control-grid columns="2" md="3">
  <nodel-button aria-label="Power">
    <nodel-icon name="power" size="md"></nodel-icon>
  </nodel-button>
  <nodel-button>
    <nodel-image src="./v2/img/logo.png" alt="Nodel" size="lg"></nodel-image>
  </nodel-button>
  <nodel-button action="SetSource" arg="HDMI1" signal="Source">
    HDMI 1
    <nodel-status-indicator signal="HDMI1Present" label="HDMI 1 signal present"></nodel-status-indicator>
  </nodel-button>
</nodel-control-grid>

Actions And Signals

Set action and optional arg to call a current-node action. A shorthand signal marks the button active when the signal value matches arg. 

<nodel-control-grid columns="3">
  <nodel-button action="SetSource" arg="Chromecast" signal="Source">Chromecast</nodel-button>
  <nodel-button action="SetSource" arg="TV" signal="Source">TV</nodel-button>
  <nodel-button action="SetSource" arg="Signage" signal="Source">Signage</nodel-button>
</nodel-control-grid>

Signal

Set signal to a local signal name to update the default target. For text, the default target is value; use signals for explicit bindings such as Status:value.

  <nodel-column md="6">
    <nodel-text signal="Status" surface="card" tone="default">Waiting for Status signal</nodel-text>
  </nodel-column>
</nodel-row>

Visibility

Use visibility on any component to show or hide it from a local signal. Values visible, true, and 1 show the component; hidden, false, and 0 hide it.

Signal-controlled panel 
<nodel-row visibility="PanelVisible">
  <nodel-column md="6">
    <nodel-text surface="card" tone="default">Signal-controlled panel</nodel-text>
  </nodel-column>
</nodel-row>

Host Icon

Use nodel-host-icon to render a generated identicon. Its default signal target is host, with explicit bindings available for link and label attributes. 

<nodel-row>
  <nodel-column md="6">
    <nodel-host-icon host="Example Host" icon-host="example-host" title="Example host"></nodel-host-icon>
  </nodel-column>
  <nodel-column md="6">
    <nodel-host-icon signal="HostName" signals="HostAddress:icon-host; HostUrl:href; HostTitle:title" host="Waiting for host signal"></nodel-host-icon>
  </nodel-column>
</nodel-row>
