Templates

Use nodel-template with a native <template> child to render repeated controls from one inert source block. Placeholders are replaced in text and attributes, so action and signal names can be generated consistently.

Repeated Controls

The rendered controls become siblings of nodel-template, which keeps them as direct children of the surrounding control grid. 

<nodel-control-grid columns="2" md="4">
  <nodel-template name="Zone" repeat="4" start="1">
    <template>
      <nodel-button join="{{item}}" variant="primary" tone="soft">
        {{name}} {{number}}
        <nodel-status-indicator signal="{{item}}Online" label="{{name}} {{number}} online"></nodel-status-indicator>
      </nodel-button>
    </template>
  </nodel-template>
</nodel-control-grid>
<nodel-template name="Level" repeat="3" data-action-prefix="SetLevel">
  <template>
    <nodel-fader label="{{name}} {{number}}" action="{{actionPrefix}}{{number}}" signal="{{item}}"></nodel-fader>
  </template>
</nodel-template>

Shared Templates

Define a native <template id="..."> once, then reference it from multiple nodel-template elements with template="id". Each use can provide its own name, repeat count, and custom data-* placeholders. 

<template id="shared-template-button">
  <nodel-button action="{{actionPrefix}}{{number}}" signal="{{item}}" variant="info" tone="outline">
    {{name}} {{number}}
  </nodel-button>
</template>

<nodel-control-grid columns="2" md="4">
  <nodel-template template="shared-template-button" name="Zone" repeat="4" data-action-prefix="SetZone"></nodel-template>
</nodel-control-grid>

<nodel-control-grid columns="2">
  <nodel-template template="shared-template-button" name="Output" repeat="2" start="5" data-action-prefix="SetOutput"></nodel-template>
</nodel-control-grid>

Faders & Meters

Use nodel-fader for touch-first level control and nodel-meter for read-only signal levels. Faders drag relative to the current value instead of jumping to a touched track position.

Vertical And Horizontal

Vertical faders are compact by default. Horizontal faders use the same range, readout, and keyboard behavior, and are best placed in a wide grid cell or full-width row. 

<nodel-control-grid columns="2">
  <nodel-fader label="Volume" value="65" nudge="5"></nodel-fader>
  <nodel-fader label="Mic" value="45"></nodel-fader>
</nodel-control-grid>

<nodel-control-grid>
  <nodel-fader label="Brightness" orientation="horizontal" value="40" increment nudge="10"></nodel-fader>
</nodel-control-grid>

dB Range

Set unit="db" for dB formatting, default -60..+10 range, and audio-style curve="vu" meter display. Min and max remain configurable. 

<nodel-control-grid columns="2">
  <nodel-fader label="Program" unit="db" value="-12" step="0.5" nudge="1" variant="warning"></nodel-fader>
  <nodel-meter unit="db" value="-6" peak="hold" readout="show" label="Program level"></nodel-meter>
</nodel-control-grid>

Meter Options

Meters can be horizontal, use custom ranges, change warning/danger thresholds, and override the display curve. 

<nodel-meter orientation="horizontal" min="0" max="10" value="7" warn="6" danger="9" readout="show" label="Temperature"></nodel-meter>
<nodel-meter unit="db" curve="linear" value="-25" readout="show" label="Linear dB override"></nodel-meter>
